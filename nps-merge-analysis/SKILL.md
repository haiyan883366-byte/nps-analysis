# NPS 两表合并 + 分析一体化 v2

输入 **NPS转化底表** + **行课底表** 两个 Excel，自动完成全流程：

```
两表合并 → 列名重命名 → 教学组匹配 → 年级筛选 → 合并底表输出 → NPS分析 → Excel(色阶) + HTML
```

## 依赖

本 skill 依赖 **nps-analysis** skill（调用其 `nps_calculator.py`）。
请确保 `nps-analysis` skill 已安装在同级目录 `~/.workbuddy/skills/nps-analysis/`。

## 输出内容（3 份文件）

| 文件 | 说明 |
|------|------|
| **{年级}合并后底表.xlsx** | 两表合并后的完整底表（含教学组/组长姓名） |
| **{年级}NPS分析表.xlsx** | NPS 汇总表，3 个 sheet（辅导维度 / 组长维度 / 汇总对比），带 ColorScale 色阶 |
| **{年级}NPS分析表.html** | HTML 报告，双维度 + 颜色标注 |

## 数据要求

### NPS转化底表

NPS 推荐值列（`nps` 或 `q1`）的值必须在 **1-11** 范围内。
如果出现 1-11 之外的数值，脚本会告警并自动剔除异常行。

| 原始列名 | 重命名为 | 说明 |
|----------|----------|------|
| q1 / nps | nps | NPS 推荐值（**1-11 分制**） |
| q10 | 是否考虑报名长期班 | 转化意向 |

同时需要以下列（用于匹配）：
- `counselor_name` — 辅导老师姓名
- `grade_names` — 年级
- `fk_course` — 课程ID

### 行课底表

需要以下列（自动检测 sheet 和列名）：
- `辅导名` — 辅导老师姓名
- `年级` — 年级
- `课程ID` — 课程ID
- `教学组` — 教学组/组长名称

## 匹配逻辑

采用 **三重键 → 双键降级** 策略解决辅导重名问题：

1. **三重键匹配**：(辅导名, 年级, 课程ID) — 精确匹配
2. **双键降级**：(辅导名, 年级) — 三重键未命中时使用
3. 匹配到的教学组填入 `组长姓名` 列，供 NPS 分析使用

## NPS 计算口径

- **NPS** = 推荐者占比 − 贬损者占比
- **推荐者**：nps >= 10 (底表1-11分制)
- **中立者**：nps 8-9
- **贬损者**：nps <= 7
- **报名率**：是否考虑报名长期班 = 1 的占比

详细口径和色阶规则参见 `nps-analysis` skill。
