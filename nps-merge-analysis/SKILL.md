# NPS 两表合并 + 分析一体化 v1

输入 **NPS转化底表** + **行课底表** 两个 Excel，自动完成全流程：

```
两表合并 → 列名重命名 → 教学组匹配 → 年级筛选 → 合并底表输出 → NPS分析 → Excel(色阶) + HTML
```

## 依赖

本 skill 依赖 **nps-analysis** skill（调用其 `nps_calculator.py`）。
请确保 `nps-analysis` skill 已安装在同级目录 `~/.workbuddy/skills/nps-analysis/`。

## 输出内容（3 份文件）

| 文件 | 说明 |
|------|------|
| **{年级}合并后底表.xlsx** | 两表合并后的完整底表（含教学组/组长姓名） |
| **{年级}NPS分析表.xlsx** | NPS 汇总表，3 个 sheet（辅导维度 / 组长维度 / 汇总对比），带 ColorScale 色阶 |
| **{年级}NPS分析表.html** | HTML 报告，双维度 + 颜色标注 |

## 数据要求

### NPS转化底表

| 原始列名 | 重命名为 | 说明 |
|----------|----------|------|
| q1 | nps | NPS 推荐值 |
| q2 | q2喜爱度 | 课程喜爱度 |
| q3 | q3收获感 | 学习收获感 |
| q4 | q4专注力 | 专注力 |
| q5 | q5表达意愿 | 表达意愿 |
| q10 | 是否考虑报名长期班 | 转化意向 |

同时需要以下列（用于匹配）：
- `counselor_name` — 辅导老师姓名
- `grade_names` — 年级
- `fk_course` — 课程ID

> 如果列名已经是目标格式（如已经是 `nps` 而非 `q1`），脚本会自动跳过重命名。

### 行课底表

需要以下列：
- `辅导名` — 辅导老师姓名
- `年级` — 年级
- `课程ID` — 课程ID
- `教学组` — 教学组/组长名称

> 脚本自动适配列名变体（如 `counselor_name` / `辅导` / `辅导名` 等）。

## 匹配逻辑

采用 **三重键 → 双键降级** 策略解决辅导重名问题：

1. **三重键匹配**：(辅导名, 年级, 课程ID) — 精确匹配
2. **双键降级**：(辅导名, 年级) — 三重键未命中时使用
3. 匹配到的教学组填入 `组长姓名` 列，供 NPS 分析使用

## 执行流程

### 1. 确认参数

- `nps_file` — NPS转化底表路径（必填）
- `xingke_file` — 行课底表路径（必填）
- `grade` — 筛选年级（如"一年级"，可选，默认全量）
- `min_samples` — 最小样本量阈值，默认 10
- `output_dir` — 输出目录（可选，默认输出到 nps_file 所在目录）

### 2. 运行脚本

```bash
python <skill-dir>/scripts/merge_and_analyze.py \
  --nps-file "<NPS转化底表.xlsx>" \
  --xingke-file "<行课底表.xlsx>" \
  --grade "一年级"
```

脚本自动完成：
1. 读取两个底表
2. 重命名 NPS转化底表列名（q1→nps 等）
3. 三重键+双键降级匹配教学组
4. 填充组长姓名列
5. 按年级筛选
6. 输出合并后底表
7. 调用 nps_calculator 生成 NPS 汇总表（3 sheet + 色阶）
8. 生成 HTML 报告

### 3. NPS 计算口径

- **NPS** = 推荐者占比 − 贬损者占比
- **推荐者**：推荐值 >= 9
- **贬损者**：推荐值 <= 6
- **报名率**：是否考虑报名长期班 = 1 的占比
- **考虑率**：= 2 的占比
- **拒绝率**：= 3 的占比

### 4. 色阶规则

| 列 | 方向 | 高值色 | 低值色 |
|----|------|--------|--------|
| NPS | 正向 | 绿 | 红 |
| 推荐率% | 正向 | 蓝 | 浅蓝 |
| 贬损率% | 反向 | 红 | 绿 |
| 报名率% | 正向 | 绿 | 橙 |
| 拒绝率% | 反向 | 红 | 绿 |

## 边界情况

### 小样本年级

当年级样本量少、辅导老师分散时，NPS 分析会自动降级：
1. 组长级汇总正常输出
2. 辅导级降低 min_samples 到 5 或 2
3. 明确告知数据局限性

### 匹配率低

如果教学组匹配率低于 80%，脚本会：
1. 列出未匹配的辅导老师名单
2. 提醒用户检查行课底表数据完整性
